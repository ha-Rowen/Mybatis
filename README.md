- 트랜잭션 기술이 많이 요구되는 금융 시스템의 송금/입금 기능을 구현합니다.

## 얻은것
- spring은 비동기 방식으로 동작하고  @Async으로 동비기 지정을 할 수 있다.

<br>

- 트랜잭션 어노테이션을 사용해서 별도의 트랜잭션을 만들었지만 <br> 완전히 분리되는 개념이 아니라서 자식메서드에 예외가 발생한다면<br> "콜스택"를 하니씩 제거하면서 호출된 메서드까지 예외가 전파된다. 
  - 위와같은 이유로 try 혹은 Async로 처리 해야한다. <br> 하지만 Async는 비동기 처리를 위한 것을 잊지말아야한다.
